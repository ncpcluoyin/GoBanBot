# GobanBot - 五子棋AI

基于 **Negamax + Alpha-Beta 剪枝** 的高性能五子棋AI，支持多线程并行搜索、彩色终端输出、人机对弈。

## ✨ 特性

- **智能决策**：深度可配置的 Negamax 搜索 + 启发式移动排序
- **多线程加速**：根节点并行，充分利用多核 CPU
- **视觉友好**：
  - 棋盘行列号紧凑显示（列号仅显示个位，如 10→0，11→1）
  - 最后一次落子高亮：🟢 玩家绿色，🔴 AI 红色
- **可调参数**：搜索深度、线程数、先后手均可运行时设置
- **高性能**：候选区域动态收缩，Alpha-Beta 剪枝减少无效分支

## 📦 编译与运行

### 依赖
- C++17 编译器（g++ 7+ 或 clang 5+）
- CMake 3.10+
- pthread 库（Linux/macOS 自带，Windows MinGW 需链接）

### 编译步骤

```bash
git clone <your-repo-url>
cd GoBanBot
mkdir build && cd build
cmake ..
make
```

### 运行

```bash
./gobanbot
```

## 🎮 使用方法

程序启动后依次询问：

1. **搜索深度**  
   推荐 `3`（普通难度）或 `4`（高难度，耗时增加）

2. **线程数量**  
   建议等于 CPU 核心数（如 `4` 或 `8`）

3. **谁先手**  
   `1` – 玩家先手 `2` – AI 先手

### 玩家输入格式

- 坐标以 **空格** 或 **逗号** 分隔，例如：
  - `7 7` （第7行第7列）
  - `7,7`
- 行列范围均为 **1~15**（对应棋盘上的编号）

### 游戏规则

- 玩家执 **X**，AI 执 **O**
- 横、竖、斜方向 **五子连珠** 即获胜
- 棋盘下满为平局

## ⚙️ 配置调优

| 参数 | 说明 | 建议值 |
|------|------|--------|
| `MAX_SEARCH_DEPTH` | 搜索深度（`negamax_ai.h` 中修改） | 3（实时） / 4（慢但强） |
| `CANDIDATE_RADIUS` | 候选点搜索半径 | 2（平衡速度与强度） |
| `DEFENSE_WEIGHT` | 防守系数（越高越倾向堵玩家） | 1.5 |
| 线程数 | 运行时输入 | CPU 核心数 |

## 🧠 AI 强度说明

- **深度 3**：普通玩家水平，能识别活三、冲四，防守及时，响应 <1 秒
- **深度 4**：较强业余段位，需等待 3~10 秒（视硬件而定）

## 🐛 常见问题

### Q: 棋盘列号显示为 `0 1 2 ... 5` 而不是 `10 11 ... 15`？
**A**: 这是设计特性，为了紧凑对齐。第10列对应数字 `0`，第11列对应 `1`，以此类推。棋盘边缘有行号辅助定位，不影响实际落子。

### Q: 高亮颜色不显示？
**A**: 需要终端支持 ANSI 转义序列。Windows 10+ 请使用 PowerShell 或新版 CMD；Linux/macOS 默认支持。

### Q: 编译报错 `unique_ptr` 或 `make_unique`？
**A**: 请确保使用 C++17 标准（CMakeLists.txt 已设置）。若手动编译添加 `-std=c++17`。
